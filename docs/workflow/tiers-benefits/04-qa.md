## Test Conventions (from codebase)

| Aspect | emf-parent UT (pointsengine-emf-ut) | emf-parent Integration | intouch-api-v3 (reference) |
|--------|--------------------------------------|------------------------|----------------------------|
| Framework | JUnit 4 (`@Test`, `@Before`, `@RunWith`) | JUnit 4, extends `BaseIntegrationTest` | JUnit 5 (`@ExtendWith(MockitoExtension.class)`) |
| Mocking | Mockito (`@Mock`, `@InjectMocks`, `MockitoAnnotations`, `MockedStatic`) | Real DB (MongoDB, MySQL) | Mockito with `@MockitoSettings(strictness = Strictness.LENIENT)` |
| Assertion | JUnit `assertEquals`, `assertNotNull`, `assertTrue` | JUnit assert | JUnit 5 `assertThrows`, `assertDoesNotThrow`, `assertEquals` |
| Naming | `test<MethodName>_<Scenario>` | `test<Scenario>` | `<methodName>_<Condition>_<Result>` |
| Location | `pointsengine-emf-ut/src/test/java/` | `integration-test/src/test/java/` | `src/test/java/` |

**Decision**: New unit tests go in `pointsengine-emf-ut`. Follow JUnit 4 + Mockito pattern matching existing codebase (per G-12.2). Integration tests go in `integration-test`.
