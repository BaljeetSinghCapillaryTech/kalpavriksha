Ticket: raidlc/ai_tier · Pipeline: feature-pipeline v1.0 · Date: 2026-04-12

Overview

New tier CRUD REST APIs in intouch-api-v3 following the dual-storage pattern (MongoDB drafts + SQL live). Includes a generic maker-checker framework (tiers are the first consumer; benefits, subscriptions plug in later). Versioned edits with parentId, soft-delete with status lifecycle, and cached member counts.

Repos: intouch-api-v3 (primary), emf-parent (Layer 3 — pending), peb (unchanged), Thrift (unchanged)

Phase 2AB — Atomic Publish (2026-04-20)

Split-brain window in TierApprovalHandler.publish() closed. The handler now does a read-modify-write on SLAB_UPGRADE + SLAB_DOWNGRADE in memory and submits slabInfo + [upgradeCopy, downgradeCopy] in a single createSlabAndUpdateStrategies Thrift transaction. All-or-nothing — engine commits slab row + both strategies + CSV auto-extension together.

Commits: 2a616290f (Thrift wrapper) · d5c226c6a (transformer helpers) · 59b4ae423 (handler rewrite). See HLD §7.5 for the end-to-end sequence diagram + CREATE/UPDATE flowchart.
