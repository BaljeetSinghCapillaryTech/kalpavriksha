## 1. US-1: Tier Listing

### Happy Path

| # | Scenario | Input | Expected | Priority |
|---|---------|-------|----------|----------|

| TC-1.2 | Filter by status | GET /v3/tiers?programId=977&status=ACTIVE | 200, only ACTIVE tiers returned | P0 |
| TC-1.3 | Filter multiple statuses | GET /v3/tiers?programId=977&status=ACTIVE,DRAFT | 200, ACTIVE and DRAFT tiers returned | P1 |
| TC-1.4 | Include inactive tiers | GET /v3/tiers?programId=977&includeInactive=true | 200, includes STOPPED tiers | P1 |
| TC-1.5 | KPI summary is correct | GET /v3/tiers?programId=977 | summary.totalTiers matches count, activeTiers matches ACTIVE count, pendingApprovalTiers matches PENDING count | P0 |

| TC-1.7 | Tiers ordered by serialNumber | GET /v3/tiers?programId=977 | serialNumber 1,2,3,4 in order | P0 |

### Edge Cases

| # | Scenario | Input | Expected | Priority |
|---|---------|-------|----------|----------|
| TC-1.8 | Empty program (no tiers) | GET /v3/tiers?programId=99999 (valid but empty) | 200, tiers: [], summary all zeros | P0 |
| TC-1.9 | Missing programId | GET /v3/tiers | 400, "programId is required" | P0 |
| TC-1.10 | Program with DRAFT + ACTIVE versions of same tier | programId has Gold (ACTIVE) + Gold Plus (DRAFT, parentId=Gold) | Both appear in listing. UI uses parentId to show version relationship. | P1 |
| TC-1.11 | Program with SNAPSHOT tiers | programId has archived versions | SNAPSHOTs excluded from default listing. Included only with includeInactive=true. | P2 |
| TC-1.12 | Multi-tenancy isolation | Org A lists tiers for programId that belongs to Org B | 200 with empty tiers (not 403 -- org filter returns no results) | P0 |

## 7. Cross-Cutting Concerns

### Multi-Tenancy (G-07)

| # | Scenario | Expected | Priority |
|---|---------|----------|----------|
| TC-7.1 | All tier queries scoped by orgId | Org A cannot see Org B's tiers even if programIds overlap | P0 |
| TC-7.2 | All pending changes scoped by orgId | Org A cannot approve Org B's pending changes | P0 |
| TC-7.3 | Thrift calls include orgId | createSlabAndUpdateStrategies passes correct orgId | P0 |

### Concurrency (@Lockable)

| # | Scenario | Expected | Priority |
|---|---------|----------|----------|
| TC-7.4 | Concurrent approvals for same program | Second approval waits or fails (not duplicate create) | P1 |
| TC-7.5 | Concurrent edits of same ACTIVE tier | Both should update the same DRAFT (not create two DRAFTs) | P1 |

### Backward Compatibility

| # | Scenario | Expected | Priority |
|---|---------|----------|----------|
| TC-7.6 | Existing PeProgramSlabDao.findByProgram() unchanged | Returns all slabs including STOPPED (no status filter) | P0 |
| TC-7.7 | New findActiveByProgram() filters correctly | Returns only ACTIVE slabs | P0 |
| TC-7.8 | PEB tier downgrade reads all slabs | PEB sees all slabs (serial number ordering intact) | P1 |

### CSV Index Correctness (R1 -- HIGH risk)

| # | Scenario | Expected | Priority |
|---|---------|----------|----------|
| TC-7.9 | Create 4th tier: threshold CSV | threshold_values has exactly 3 values (N-1 for N slabs) | P0 |
| TC-7.10 | Create 5th tier: threshold CSV | threshold_values has exactly 4 values | P0 |
| TC-7.11 | Edit 3rd tier threshold: CSV position | Position 1 (serialNumber 3 - 2 = 1) in CSV updated. Others unchanged. | P0 |
| TC-7.12 | Allocation CSV auto-extended | POINT_ALLOCATION allocation_values has N values after creating Nth slab | P0 |
| TC-7.13 | Expiry CSV auto-extended | POINT_EXPIRY expiry_time_values has N values after creating Nth slab | P1 |

## 8. Test Data Requirements

### Programs
- Program 977 (4 slabs: Bronze/Silver/Gold + Platinum draft)
- Program with MC enabled
- Program with MC disabled

- Base tier (serialNumber=1, no downgrade)
- Mid tier (serialNumber=2, threshold=550)

- DRAFT tier (not yet approved)
- PENDING_APPROVAL tier

- Tier with PartnerProgramSlab references

### Strategies
- SLAB_UPGRADE strategy with threshold CSV
- SLAB_DOWNGRADE strategy with TierConfiguration JSON
- POINT_ALLOCATION strategies (for CSV extension verification)
- POINT_EXPIRY strategies (for CSV extension verification)
