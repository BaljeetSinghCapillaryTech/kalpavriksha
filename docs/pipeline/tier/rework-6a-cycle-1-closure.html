Overview

Rework #6a is a contract-hardening cycle — not a feature change. Zero schema changes, zero Thrift changes, zero engine changes. The work converts silent Jackson-drop and permissive-field-accept behaviour on the tier write path into a deterministic, traceable pre-binding rejection contract with 8 dedicated error codes (9011–9018) and an asymmetric read/write envelope.

Motivation: The UI team's v3 tier integration exposed that the existing v3.1 validator silently accepted ignored fields (e.g. legacy downgrade objects from v2 round-trips) and used opaque generic 400s when rejecting. The new contract surfaces precise error codes, rejects pre-binding (before Jackson sees the payload), and documents the write-narrow / read-wide asymmetry.

Primary repo: intouch-api-v3 (branch common-sprint-73). Orchestrator: kalpavriksha (branch raidlc/ai_tier).

Scope & Non-scope

In Scope

  • Contract rename: legacy downgrade block removed from write DTOs
  • 8 new error codes 9011–9018 with deterministic scan ordering
  • Pre-binding recursive scanners for Class A (program-level orchestration flags) and Class B (schedule/nudges/notificationConfig)
  • Wire contract: [9011] bracket-prefix → errors[0].code numeric via TargetGroupErrorAdvice regex extractor
  • Asymmetric envelope: write-narrow request DTOs, read-wide response from engine
  • Tier-scoped Jackson strict-mode via @JsonIgnoreProperties(ignoreUnknown = false) (Phase 11 R11-2 (b+c))

Explicitly Out of Scope

  • ❌ Database schema changes
  • ❌ Thrift IDL or protocol changes
  • ❌ Engine-side tier/slab strategy changes
  • ❌ Global ObjectMapper bean hardening (risk to 35+ opt-out DTOs)
  • ❌ Benefits / subscriptions / program-level rework

R11-2 Resolution — Tier-scoped Jackson Hardening + BT-197b

Problem

ADR-20R's write-narrow pre-binding scan contract relies on Jackson rejecting unknown keys (i.e. unclassified-but-unwanted fields) via FAIL_ON_UNKNOWN_PROPERTIES. The @Bean ObjectMapper in IntouchApiV3Application does not explicitly configure this — it relies on Jackson's native default (true), which Spring Boot's global property spring.jackson.deserialization.fail-on-unknown-properties can override. If anyone flips this property (or refactors to use Jackson2ObjectMapperBuilder), the write-narrow guarantee silently breaks for any key not covered by the pre-binding scanners.

Resolution: (b) Tier-scoped annotation

Option (b) + (c) chosen over (a) (global hardening) to avoid blast-radius on 35+ DTOs that opt out of strict-mode intentionally. Tier-scoped annotation cannot be overridden by the global property.
