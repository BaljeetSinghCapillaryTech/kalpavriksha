Accepted Deviations

Four deviations were formally accepted with documented rationale. Each is tracked in session-memory and must be reviewed at the trigger conditions listed.

DecisionDeviationRationaleRevisit Triggers
D-24 / ADR-008 G-01.3 prohibits java.util.Date in DTOs; this feature uses it to match platform convention (Benefits.java P-01 exemplar) G-12.2 compliance (match existing pattern) takes priority at this layer; ISO-8601 UTC contract is correctly enforced at the REST boundary via @JsonFormat Migration of entity to MongoDB or adoption of java.time across emf-parent
D-61 / F-01 ADR-002/ADR-004 specify @PatchMapping for /activate and /deactivate; implementation uses @PostMapping Platform intouch-api-v3 convention for state-change operations uses POST; changing would break Spring MVC routing contract in production; semantic difference has no runtime impact Platform-wide migration to PATCH for state-change verbs
D-62 / F-03 ADR-011 specifies max 100 on list size with controller @Max(100) gate; controller gate was NOT applied — service-layer BC_PAGE_SIZE_EXCEEDED throw only User directive: controller-level @Max annotation generates additional @Valid chain complexity; service throw at size>100 gives equivalent behaviour; @Max import already removed from CreateRequest (R-02) Client-visible 400 response shape changes; API spec published to consumers
D-63 / R-03 ADR-009 + Designer §E specify HTTP 409 for BC_UNKNOWN_SLAB / BC_CROSS_PROGRAM_SLAB; implementation throws 400 for slab-not-in-program validation 400 is semantically correct for "bad request payload" when slabId is unknown; 409 is reserved for state conflicts; team accepted 400 as more accurate; ADR-009 error taxonomy updated in session-memory API consumers report unexpected 400 on cross-program slab errors; error taxonomy formal review
