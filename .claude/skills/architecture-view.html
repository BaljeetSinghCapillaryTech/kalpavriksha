3. Architect-Analyst-ProductEx Verification Cycle

The 3-cycle max loop ensuring Architect's solution fulfils product requirements

flowchart TB
    ARCH["Architect produces\n01-architect.md"] --> U1{"User\napproves?"}
    U1 -->|continue| STAGE1

    subgraph ANALYST_WORK["Analyst Phase 02"]
        STAGE1["Stage 1: Impact Analysis\nblast radius, security, risks"]
        STAGE1 --> STAGE2["Stage 2: Spawn\nProductEx verify"]
    end

    STAGE2 --> VERDICT{"ProductEx\nverdict?"}

    VERDICT -->|approved| OK["Analyst returns\ncomplete + verification"]
    VERDICT -->|changes needed| BLOCK["Analyst raises BLOCKER\nImpact + Product issues\nwith evidence chain"]

    OK --> SHOW_OK["Show user:\nfindings + approval"] --> NEXT["Proceed to Designer"]

    BLOCK --> SHOW["Show user: Cycle N/3\nIssues + Evidence\n+ Suggested directions"]

    SHOW --> CK{"Cycle\ncount?"}

    CK -->|"< 3"| APPROVE{"User approves\nrework?"}
    APPROVE -->|continue| RERUN["Architect re-runs\nwith evidence + directions"]
    APPROVE -->|resolve| MANUAL["Human takes\nmanual control"]
    RERUN --> ARCH

    CK -->|"= 3"| ESC["HUMAN ESCALATION\n3 cycles exhausted"]
    ESC --> OPT["Options:\nA. Manual adjust\nB. Accept with risks\nC. Revisit BA\nD. Consult product team"]
    OPT --> DECIDE["Human decides"] --> CONT["Continue workflow"]

6. Blocker & Rework Loop

How blockers are classified, cascade rules, and circuit breaker logic

flowchart TB
    BL["Phase X raises BLOCKER\nTARGET = Phase N"] --> CHK{"Rework Log:\nPhase N re-run\ncount?"}

    CHK -->|">= 2"| CB["CIRCUIT BREAKER\nEscalate to Human"]
    CHK -->|"< 2"| SEV{"Orchestrator\nclassifies severity"}

    SEV -->|trivial| TRV["Auto re-run Phase N\nwith blocker appended"]
    SEV -->|critical| CRT["Pause for\nuser approval"]

    CRT --> UA["User approves"] --> RR["Re-run Phase N"]
    TRV --> CAS["Cascade through\nintermediate phases"]
    RR --> PC["Pause: cascade?"] --> CAS

    CAS --> UL["Update Rework Log"] --> RET["Return to Phase X"]

    CB --> OPT["Options:\nA. Revisit earlier phase\nB. Accept deviation\nC. Mark as risk\nD. Other"]
